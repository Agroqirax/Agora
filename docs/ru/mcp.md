# MCP-серверов

Agora может подключаться к серверам [Model Context Protocol](https://modelcontextprotocol.io) и позволять модели вызывать инструменты, которые они предоставляют — поисковые системы, базы данных, домашнюю автоматизацию, внутренние API компании или что-либо еще, для чего вы или третья сторона создали сервер MCP.

!!! Примечание
    В настоящее время Agora поддерживает только **инструменты** MCP. Ресурсы, подсказки и выборка еще не реализованы.

## Как это работает

```текст
Agora (Android) ──HTTPS (потоковая передача HTTP)──▶ Сервер MCP
                                                          │
                                                          ├── инициализировать
                                                          ├── инструменты/список
                                                          └── инструменты/вызов
```

Agora использует транспорт MCP **Streamable HTTP** (одну конечную точку HTTP, а не stdio или более старый транспорт HTTP+SSE). При первом использовании он открывает сеанс с помощью `initialize`, выводит список инструментов сервера с помощью `tools/list` и позволяет модели вызывать их с помощью `tools/call`. Список инструментов кэшируется примерно на 30 секунд на каждом сервере, поэтому повторяющиеся сообщения не требуют повторного подтверждения каждый раз; если сервер завершает сеанс, Agora автоматически подключается повторно при следующем вызове.

Модель сама решает, когда использовать инструмент MCP, точно так же, как она решает использовать веб-поиск или оболочку — ручного запуска нет.

## Безопасность

Серверы MCP, которые вы добавляете, представляют собой произвольный код, которому вы доверяете доступ к инструментам, поэтому Agora по умолчанию обращается с ними осторожно:

- **Инструменты только для чтения запускаются без запроса.** Если сервер помечает инструмент как `readOnlyHint`, Agora вызывает его автоматически.
- **Все остальное требует подтверждения.** Если инструмент не помечен как доступный только для чтения, Agora рассматривает его как потенциально деструктивный (включая инструменты, которые вообще не объявляют `destructiveHint`) и перед его запуском показывает диалоговое окно подтверждения с именем инструмента и аргументами.
- **"Всегда разрешать этот сервер"** позволяет пропустить запрос до конца сеанса. Это сбрасывается при перезапуске Агоры.
- **Аутентификация отправляется только на настроенный вами сервер.** Добавляемый вами токен носителя или пользовательский заголовок отправляется исключительно на URL-адрес этого сервера.

!!! предупреждение
    Если URL-адрес сервера использует простой `http://` вместо `https://`, любой токен носителя или заголовок, который вы настраиваете, передается незашифрованным. Отдавайте предпочтение конечным точкам https://, особенно в ненадежных сетях.

## Настройка

### Шаг 1. Получите сервер MCP

Это может быть общедоступный сервер MCP, который работает внутри вашей организации или который вы размещаете самостоятельно. Он должен предоставлять транспорт **Streamable HTTP** по одному URL-адресу (обычно заканчивающемуся на `/mcp`).

### Шаг 2: Добавьте это в Agora

1. Перейдите в **Настройки → Серверы MCP**.
2. Включите **Включить инструменты MCP**.
3. Нажмите **Добавить сервер**.
4. Заполните данные сервера:

| Поле | Описание | Пример |
| ----------------- | ----------------------------------------------------------------------------------------------------- | --------------------------------- |
| **Имя** | Отображаемое имя этого сервера | `Домашний помощник` |
| **Описание** | Необязательное примечание о том, для чего это нужно. Если оставить пустым, вместо этого отображается хост сервера.                 | `Управляет освещением и термостатами` |
| **URL-адрес сервера** | Конечная точка HTTP MCP Streamable | `https://example.com/mcp` |
| **Жетон на предъявителя** | Необязательно — отправляется как `Авторизация: Носитель <токен>` | Токен API вашего сервера |
| **Дополнительные заголовки** | Необязательно — по одному в строке, как «Имя: значение», для серверов, которые ожидают аутентификацию или маршрутизацию по пользовательскому заголовку | `X-Api-Key: секрет` |
| **Тайм-аут** | Таймаут для каждого запроса, 5–120 секунд | `30` |

5. Нажмите **Проверить соединение**, чтобы убедиться, что Agora может подключиться к серверу, и посмотреть, сколько инструментов он предоставляет, прежде чем сохранять.
6. Нажмите **Сохранить**.

После успешного подключения сервера — либо в результате тестового соединения, либо в результате фактического использования в чате — его имя и версия (сообщенные сервером) отображаются в виде небольшого значка рядом с его записью в списке серверов.

### Шаг 3: Используйте

Отправьте сообщение, которое может использовать один из инструментов сервера. Если инструмент не доступен только для чтения, вам будет предложено разрешить его в первый раз; после этого оно сохраняется до конца сеанса (или до тех пор, пока вы его не откажете).

## Поддержка нескольких серверов

Добавьте столько серверов, сколько захотите — API поиска, внутреннюю систему заявок, центр домашней автоматизации. Каждый из них настраивается и аутентифицируется независимо, а их инструменты автоматически распределяются по пространству имен (например, `mcp__home_assistant__turn_on_light`), поэтому инструменты с одинаковыми именами на разных серверах никогда не конфликтуют.

Отключение **Включить инструменты MCP** или флажка **Включено** отдельного сервера удаляет его инструменты из области видимости модели, не удаляя при этом его конфигурацию.

## Подтверждение вызовов деструктивных инструментов

Вы можете полностью отключить запросы на подтверждение с помощью **Подтверждать деструктивные вызовы инструментов MCP** в **Настройки → Серверы MCP** — инструменты только для чтения всегда запускаются без запроса, независимо от этого параметра. Отключение означает, что каждый инструмент на каждом включенном сервере запускается немедленно, без запроса, поэтому отключайте его только для серверов, которым вы полностью доверяете.

## Устранение неполадок

### Тестовое соединение не выполнено

– Дважды проверьте **URL-адрес сервера** — это должна быть полная конечная точка (например, `.../mcp`), а не только хост.
– Если сервер требует аутентификации, проверьте правильность **Токена носителя** или **Дополнительных заголовков**.
– Убедитесь, что сервер реализует потоковый транспорт HTTP, а не stdio или устаревший транспорт HTTP+SSE.
– Убедитесь, что URL-адрес доступен с вашего устройства (а не только из сети вашего рабочего стола).

### Модель никогда не вызывает инструмент

- Убедитесь, что флажок **Включить MCP Tools** и флажок **Включено** для конкретного сервера включены.
– Попробуйте **Проверить соединение**, чтобы убедиться, что на сервере в настоящее время указан этот инструмент.
- Некоторые модели более неохотно вызывают в разговоре незнакомые инструменты без четкой причины — постарайтесь четко сказать, что вы хотите сделать.

### Запросы продолжают терять время

– Увеличьте **Таймаут** сервера, если его инструменты работают медленно (например, длительный поиск или автоматизация).
- Медленный или перегруженный сервер повторяет попытку не чаще одного раза в 30 секунд, а не при каждом сообщении, поэтому временный сбой не будет постоянно останавливать ваш разговор.

### В запросе подтверждения отображаются неожиданные аргументы

Диалоговое окно подтверждения показывает точные аргументы, которые модель собирается отправить. Если они кажутся неправильными, отклоните вызов — модель обычно увидит отказ и скорректирует следующую попытку.
