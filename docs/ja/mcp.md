MCP サーバーの数

Agora は [モデル コンテキスト プロトコル](https://modelcontextprotocol.io) サーバーに接続し、モデルが公開するツール (検索エンジン、データベース、ホーム オートメーション、社内 API、またはユーザーまたはサードパーティが MCP サーバーを構築したその他のもの) を呼び出すことができます。

!!!メモ
    Agora は現在、MCP **ツール** のみをサポートしています。リソース、プロンプト、サンプリングはまだ実装されていません。

## 仕組み

```テキスト
Agora (Android) ──HTTPS (ストリーミング可能な HTTP トランスポート)──▶ MCP サーバー
                                                          │
                                                          §── 初期化する
                                                          §── ツール/リスト
                                                          └── ツール/コール
「」

Agora は、MCP **ストリーミング可能な HTTP** トランスポート (標準入出力や古い HTTP+SSE トランスポートではなく、単一の HTTP エンドポイント) を通信します。初めて使用するとき、`initialize` でセッションを開き、`tools/list` でサーバーのツールをリストし、`tools/call` でモデルにそれらを呼び出します。ツール リストはサーバーごとに約 30 秒間キャッシュされるため、繰り返されるメッセージが毎回再ハンドシェイクされることはありません。サーバーがセッションを終了すると、Agora は次の呼び出し時に自動的に再接続します。

モデルは、Web 検索やシェルの使用を決定するのと同じ方法で、MCP ツールをいつ使用するかを独自に決定します。手動トリガーはありません。

## セキュリティ

追加する MCP サーバーは、ツールへのアクセスを信頼するために選択した任意のコードであるため、Agora はデフォルトでそれらを慎重に扱います。

- **読み取り専用ツールは何も要求せずに実行されます。** サーバーがツールに `readOnlyHint` をマークすると、Agora はそのツールを自動的に呼び出します。
- **それ以外はすべて確認を求めます。** ツールが読み取り専用としてマークされていない場合、Agora はそのツールを潜在的に破壊的なものとして扱います (単に `destructiveHint` をまったく宣言していないツールも含む)、実行前にツール名と引数を含む確認ダイアログを表示します。
- **「常にこのサーバーを許可する」** を選択すると、残りのセッションではプロンプトをスキップできます。これは、Agora が再起動されるとリセットされます。
- **認証は、設定したサーバーにのみ送信されます。** 追加したベアラー トークンまたはカスタム ヘッダーは、そのサーバーの URL にのみ送信されます。

!!!警告
    サーバーの URL が「https://」ではなくプレーンな「http://」を使用している場合、設定したベアラー トークンまたはヘッダーは暗号化されずに送信されます。特に信頼できないネットワークでは、「https://」エンドポイントを優先します。

## セットアップ

### ステップ 1: MCP サーバーを取得する

これは、パブリック MCP サーバー、組織が内部で実行するサーバー、または自分でホストするサーバーのいずれかにすることができます。 **ストリーミング可能な HTTP** トランスポートを単一の URL (通常は `/mcp` で終わる) で公開する必要があります。

### ステップ 2: Agora に追加する

1. **[設定] → [MCP サーバー]** に移動します。
2. **MCP ツールを有効にする** を有効にします
3. [**サーバーの追加**] をタップします
4. サーバーの詳細を入力します。

|フィールド |説明 |例 |
| ----------------- | ----------------------------------------------------------------------------------------------------- | --------------------------------- |
| **名前** |このサーバーの表示名 | 「ホームアシスタント」 |
| **説明** |目的に関するメモ (オプション)。空白のままにすると、代わりにサーバーのホストが表示されます。                 | `ライトとサーモスタットを制御` |
| **サーバー URL** | MCP ストリーミング可能な HTTP エンドポイント | `https://example.com/mcp` |
| **ベアラートークン** |オプション — `Authorization: Bearer <token>` として送信されます。サーバーの API トークン |
| **追加ヘッダー** |オプション — カスタム ヘッダーによる認証またはルーティングを期待するサーバーの場合は、「名前: 値」として 1 行に 1 つ。 `X-Api-Key: シークレット` |
| **タイムアウト** |リクエストごとのタイムアウト、5 ～ 120 秒 | `30` |

5. [**接続テスト**] をタップして、Agora がサーバーにアクセスできることを確認し、保存する前に公開されているツールの数を確認します。
6. [**保存**] をタップします。

テスト接続またはチャットでの実際の使用によってサーバーが正常に接続されると、その名前とバージョン (サーバーによって報告される) がサーバー リストのエントリの横に小さなバッジとして表示されます。

### ステップ 3: 使用する

サーバーのツールのいずれかを使用できるメッセージを送信します。ツールが読み取り専用でない場合は、最初にツールを許可するように求められます。その後、セッションの残りの間 (または拒否するまで) 記憶されます。

## マルチサーバーのサポート

検索 API、内部発券システム、ホーム オートメーション ハブなど、サーバーを好きなだけ追加できます。それぞれは独立して構成および認証され、ツールには自動的に名前空間が割り当てられるため (例: 「mcp__home_assistant__turn_on_light」)、異なるサーバーの同じ名前のツールが衝突することはありません。

[**MCP ツールを有効にする**] または単一サーバー独自の [**有効**] チェックボックスを無効にすると、構成を削除せずに、モデルが表示できるツールがモデルから削除されます。

## 破壊的なツールの呼び出しを確認する

**[設定] → [MCP サーバー]** の [**破壊的な MCP ツール呼び出しの確認**] を使用して、確認プロンプトを完全にオフにすることができます。読み取り専用ツールは、この設定に関係なく、常に確認せずに実行されます。これをオフにすると、有効なすべてのサーバーのすべてのツールがプロンプトなしですぐに実行されるため、完全に信頼できるサーバーに対してのみ無効にします。

## トラブルシューティング

### テスト接続が失敗する

- **サーバー URL** を再確認します。ホストだけではなく、完全なエンドポイント (例: `.../mcp`) である必要があります。
- サーバーに認証が必要な場合は、**ベアラー トークン** または **追加ヘッダー** が正しいことを確認してください。
- サーバーが stdio または従来の HTTP+SSE トランスポートではなく、Streamable HTTP トランスポートを実装していることを確認します。
- URL がデバイス (デスクトップのネットワークだけでなく) から到達可能であることを確認します。

### モデルがツールを呼び出すことはありません

- **MCP ツールを有効にする** と特定のサーバーの **有効** チェックボックスが両方ともオンになっていることを確認します
- **テスト接続**を試して、サーバーが現在そのツールをリストしていることを確認します
- 一部のモデルは、会話の中で明確な理由がなければ、なじみのないツールを呼び出すことに消極的です。何をしてほしいかを明確にするようにしてください。

### リクエストがタイムアウトし続ける

- サーバーのツールが遅い場合 (長時間実行される検索や自動化など)、サーバーの **タイムアウト** を増やします。
- 低速または過負荷のサーバーは、メッセージごとではなく、最大 30 秒に 1 回再試行されるため、一時的な停止によって会話が繰り返し停止することはありません

### 確認プロンプトに予期しない引数が表示される

確認ダイアログには、モデルが送信しようとしている正確な引数が表示されます。間違っていると思われる場合は、呼び出しを拒否します。通常、モデルは拒否を確認して、次の試行を調整します。
