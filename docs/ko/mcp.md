# MCP 서버

Agora는 [모델 컨텍스트 프로토콜](https://modelcontextprotocol.io) 서버에 연결하여 검색 엔진, 데이터베이스, 홈 자동화, 회사 내부 API 또는 귀하 또는 제3자가 MCP 서버를 구축한 기타 모든 도구를 모델이 호출하도록 할 수 있습니다.

!!! 참고
    Agora는 현재 MCP **도구**만 지원합니다. 리소스, 프롬프트 및 샘플링은 아직 구현되지 않았습니다.

## 작동 방식

``텍스트
Agora (Android) ──HTTPS (스트리밍 가능한 HTTP 전송)──▶ MCP 서버
                                                          │
                                                          ├── 초기화
                                                          ├── 도구/목록
                                                          └── 도구/통화
````

Agora는 MCP **스트리밍 가능한 HTTP** 전송(stdio 또는 이전 HTTP+SSE 전송이 아닌 단일 HTTP 엔드포인트)을 사용합니다. 처음 사용할 때 'initialize'를 사용하여 세션을 열고, 'tools/list'를 사용하여 서버의 도구를 나열하고, 모델이 'tools/call'을 사용하여 해당 도구를 호출할 수 있도록 합니다. 도구 목록은 서버당 약 30초 동안 캐시되므로 반복되는 메시지가 매번 다시 핸드셰이크되지는 않습니다. 서버가 세션을 종료하면 Agora는 다음 호출에서 자동으로 다시 연결합니다.

모델은 웹 검색이나 셸을 사용하기로 결정하는 것과 동일한 방식으로 MCP 도구를 자체적으로 사용할 시기를 결정합니다. 수동 트리거가 없습니다.

## 보안

추가하는 MCP 서버는 도구 액세스를 신뢰하기로 선택한 임의의 코드이므로 Agora는 기본적으로 이를 신중하게 처리합니다.

- **읽기 전용 도구는 묻지 않고 실행됩니다.** 서버가 'readOnlyHint'로 도구를 표시하면 Agora는 자동으로 이를 호출합니다.
- **다른 모든 것에서는 확인을 요청합니다.** 도구가 읽기 전용으로 표시되지 않은 경우 Agora는 해당 도구를 잠재적으로 파괴적인 것으로 간주합니다('destructiveHint'를 전혀 선언하지 않는 도구 포함). 그리고 도구를 실행하기 전에 도구 이름과 인수가 포함된 확인 대화 상자를 표시합니다.
- **"항상 이 서버 허용"**을 사용하면 나머지 세션 동안 프롬프트를 건너뛸 수 있습니다. Agora가 다시 시작되면 재설정됩니다.
- **인증은 귀하가 구성한 서버로만 전송됩니다.** 귀하가 추가한 Bearer 토큰 또는 사용자 정의 헤더는 해당 서버의 URL로만 전송됩니다.

!!! 경고
    서버의 URL이 `https://` 대신 일반 `http://`를 사용하는 경우 구성한 모든 Bearer 토큰 또는 헤더는 암호화되지 않은 상태로 이동합니다. 특히 신뢰할 수 없는 네트워크에서는 `https://` 엔드포인트를 선호합니다.

## 설정

### 1단계: MCP 서버 확보

이는 공개 MCP 서버, 조직이 내부적으로 실행하는 서버 또는 직접 호스팅하는 서버일 수 있습니다. 단일 URL(일반적으로 `/mcp`로 끝남)에서 **스트리밍 가능한 HTTP** 전송을 노출해야 합니다.

### 2단계: 아고라에 추가

1. **설정 → MCP 서버**로 이동합니다.
2. **MCP 도구 활성화**를 활성화합니다.
3. **서버 추가**를 탭합니다.
4. 서버 세부 정보를 입력합니다.

| 필드 | 설명 | 예 |
| ----------------- | ---------------------------------------------------------------------------------------- | --------------------------------- |
| **이름** | 이 서버의 표시 이름 | '홈어시스턴트' |
| **설명** | 용도에 대한 선택적 메모입니다. 공백으로 두면 서버의 호스트가 대신 표시됩니다.                 | '조명 및 온도 조절 장치 제어' |
| **서버 URL** | MCP 스트리밍 가능 HTTP 엔드포인트 | `https://example.com/mcp` |
| **베어러 토큰** | 선택 사항 - `Authorization: Bearer <token>`으로 전송됨 | 서버의 API 토큰 |
| **추가 헤더** | 선택 사항 - 사용자 정의 헤더에 의한 인증 또는 라우팅이 필요한 서버의 경우 '이름: 값'으로 한 줄에 하나씩 | `X-Api-Key: 비밀` |
| **시간 초과** | 요청별 제한 시간, 5~120초 | `30` |

5. 저장하기 전에 **연결 테스트**를 눌러 Agora가 서버에 연결할 수 있는지 확인하고 노출되는 도구 수를 확인하세요.
6. **저장**을 탭합니다.

연결 테스트 또는 채팅의 실제 사용을 통해 서버가 성공적으로 연결되면 해당 이름과 버전(서버에서 보고한 대로)이 서버 목록 항목 옆에 작은 배지로 표시됩니다.

### 3단계: 사용

서버 도구 중 하나를 사용할 수 있는 메시지를 보냅니다. 도구가 읽기 전용이 아닌 경우 처음에 이를 허용하라는 메시지가 표시됩니다. 그 후에는 나머지 세션 동안(또는 거부할 때까지) 기억됩니다.

## 다중 서버 지원

검색 API, 내부 티켓 시스템, 홈 자동화 허브 등 원하는 만큼 서버를 추가하세요. 각각은 독립적으로 구성 및 인증되며 해당 도구는 자동으로 네임스페이스가 지정되므로(예: `mcp__home_assistant__turn_on_light`) 서로 다른 서버의 동일한 이름을 가진 도구는 충돌하지 않습니다.

**MCP 도구 활성화** 또는 단일 서버 자체의 **활성화** 확인란을 비활성화하면 해당 구성을 삭제하지 않고도 모델이 볼 수 있는 도구에서 해당 도구가 제거됩니다.

## 파괴적인 도구 호출 확인

**설정 → MCP 서버**에서 **파괴적인 MCP 도구 호출 확인**을 사용하여 확인 메시지를 완전히 끌 수 있습니다. 읽기 전용 도구는 이 설정에 관계없이 항상 묻지 않고 실행됩니다. 이 기능을 끄면 활성화된 모든 서버의 모든 도구가 메시지 없이 즉시 실행되므로 완전히 신뢰하는 서버에 대해서만 비활성화하세요.

## 문제 해결

### 연결 테스트 실패

- **서버 URL**을 다시 확인하세요. 호스트뿐만 아니라 전체 엔드포인트(예: `.../mcp`)여야 합니다.
- 서버에 인증이 필요한 경우 **Bearer Token** 또는 **Extra Headers**가 올바른지 확인하세요.
- 서버가 stdio 또는 레거시 HTTP+SSE 전송이 아닌 스트리밍 가능한 HTTP 전송을 구현하는지 확인하세요.
- 데스크톱 네트워크뿐만 아니라 기기에서 URL에 연결할 수 있는지 확인하세요.

### 모델은 도구를 호출하지 않습니다.

- **MCP 도구 활성화**와 특정 서버의 **활성화** 확인란이 모두 켜져 있는지 확인합니다.
- **연결 테스트**를 시도하여 현재 서버에 해당 도구가 나열되어 있는지 확인하세요.
- 일부 모델은 대화에서 명확한 이유 없이 익숙하지 않은 도구를 호출하는 것을 더 꺼립니다. 수행하려는 작업에 대해 명시적으로 설명해보세요.

### 요청 시간이 계속 초과됩니다.

- 도구가 느린 경우(예: 장기 실행 검색 또는 자동화) 서버의 **시간 초과**를 늘립니다.
- 느리거나 과부하된 서버는 모든 메시지가 아닌 최대 30초마다 다시 시도되므로 일시적인 중단으로 인해 대화가 반복적으로 중단되지 않습니다.

### 확인 프롬프트에 예상치 못한 인수가 표시됩니다.

확인 대화 상자에는 모델이 보내려는 정확한 인수가 표시됩니다. 잘못된 것으로 보이면 호출을 거부합니다. 모델은 일반적으로 거부를 확인하고 다음 시도를 조정합니다.
